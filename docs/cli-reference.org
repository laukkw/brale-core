** 概述

=bralectl= 是 brale-core 量化交易引擎的命令行管理工具。它通过 Runtime API（默认
=http://127.0.0.1:9991=）连接运行中的 brale-core 实例，执行运维操作；同时也支持
离线操作——回测（Backtest）、指标对比（Indicator Diff）、MCP 服务——只需配置文件
和本地数据库。

** 全局参数

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--endpoint= | =http://127.0.0.1:9991= | Runtime API 地址 |
| =--json= | =false= | 以 JSON 格式输出 |

** 命令

*** llm — LLM 诊断

探测当前配置的 LLM 端点是否支持 structured output。

#+begin_src bash
bralectl llm probe [--repo .] [--stage indicator|structure|mechanics]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--repo= | =.= | 项目根目录路径，用于读取配置 |
| =--stage= | （空，探测全部） | 只探测单个 stage：=indicator=, =structure=, =mechanics= |

输出示例：

#+begin_src text
indicator   OK    model=gpt-4o   endpoint=https://api.openai.com/v1
structure   OK    model=gpt-4o   endpoint=https://api.openai.com/v1
mechanics   FAIL  model=gpt-4o   endpoint=https://api.openai.com/v1  err=timeout
#+end_src

评估最近的 LLM rounds 元数据质量：

#+begin_src bash
bralectl llm eval [--symbol BTCUSDT] [--limit 20] [--db postgres://...]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--symbol= | （空） | 只评估指定交易对 |
| =--limit= | =20= | 最多评估多少条 rounds |
| =--db= | （空） | PostgreSQL DSN |

查看 prompt registry：

#+begin_src bash
bralectl llm prompts list [--role decision] [--active-only] [--db postgres://...]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--role= | （空） | 按角色过滤 prompt |
| =--active-only= | =false= | 只显示激活中的 prompt |
| =--db= | （空） | PostgreSQL DSN |

*** add-symbol — 添加交易币种

从现有币种模板生成配置文件，注册新的交易对。

#+begin_src bash
bralectl add-symbol ETHUSDT [--repo .] [--template-symbol BTCUSDT] [--dry-run]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =SYMBOL= | （必填，位置参数） | 要添加的交易对，如 =ETHUSDT= |
| =--repo= | =.= | 项目根目录路径 |
| =--template-symbol= | （空，交互选择） | 模板币种；为空时进入交互选择 |
| =--dry-run= | =false= | 只预览生成内容，不写入文件 |

*** setup — 本地环境与 MCP 安装向导

初始化 =.env=（若缺失则从 =.env.example= 复制），并可选写入 MCP 客户端配置。

#+begin_src bash
bralectl setup [--repo .] [--lang zh|en] [--install-mcp] [--skip-mcp] \
  [--mcp-target claude-code|claude-desktop|opencode|codex|custom] [--config PATH]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--repo= | =.= | 项目根目录路径 |
| =--lang= | 交互选择（非交互默认 =en=） | 向导语言 |
| =--install-mcp= | =false= | 直接执行 MCP 安装流程 |
| =--skip-mcp= | =false= | 跳过 MCP 安装流程 |
| =--mcp-target= | =claude-code= | MCP 客户端目标：=claude-code=、=claude-desktop=、=opencode=、=codex= 或 =custom= |
| =--config= | （空） | 显式指定 MCP 配置文件路径；当 =--mcp-target= =custom= 时必填 |
| =--command= | （自动检测） | =bralectl= 可执行文件路径 |
| =--name= | =brale-core= | MCP server 名称 |
| =--system= | =<repo>/configs/system.toml= | system.toml 路径 |
| =--index= | =<repo>/configs/symbols-index.toml= | symbols-index.toml 路径 |
| =--audit-log= | （自动生成） | MCP 审计日志路径 |

*** observe — 市场观察

| 子命令 | 说明 |
|--------+------|
| =observe run --symbol SYM [--yes]= | 手动触发一次观察轮次。=--yes= 跳过确认提示。 |
| =observe report --symbol SYM [--json]= | 查看最近一次观察报告（Markdown 格式）。 |

*** indicator — 指标引擎工具

对比不同指标引擎的输出差异。可选引擎：=ta=（默认，纯 Go）、=talib=（CGO）、=reference=（参考实现）。

#+begin_src bash
bralectl indicator diff --symbol BTCUSDT \
  [--index PATH] [--baseline talib] [--candidate ta] [--format text|json]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--symbol= | （必填） | 交易对 |
| =--index= | =configs/symbols-index.toml= | symbols-index.toml 路径 |
| =--baseline= | =talib= | 基准引擎：=ta=、=talib= 或 =reference= |
| =--candidate= | =ta= | 对比引擎：=ta=、=talib= 或 =reference= |
| =--format= | =text= | 输出格式：=text= 或 =json= |

*** backtest — Gate 规则回放

将历史决策轮次重放进当前 Gate/Sieve 规则引擎，无需调用 LLM。

#+begin_src bash
bralectl backtest rules --symbol BTCUSDT \
  --from 2025-01-01 --to 2025-06-01 \
  [--format text|json|html] [--output report.html] \
  [--system PATH] [--index PATH] [--db PATH]
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--symbol= | （必填） | 交易对 |
| =--from= / =--to= | （必填） | 日期范围，支持 =YYYY-MM-DD= 或 RFC 3339 |
| =--system= | =configs/system.toml= | system.toml 路径 |
| =--index= | =configs/symbols-index.toml= | symbols-index.toml 路径 |
| =--db= | 从 system.toml 读取 | PostgreSQL DSN |
| =--format= | =text= | 输出格式：=text=, =json=, =html= |
| =--output= | stdout | 报告输出文件路径（=html= 格式必须指定） |

*** memory — 记忆系统管理

管理语义规则（Semantic Rule）存储，查看情景式交易反思（Episodic Reflection）。
所有 memory 子命令支持 =--db= 参数指定 PostgreSQL DSN。
当前未提供 =--system=；留空时会回退到本地默认 DSN。

| 子命令 | 说明 |
|--------+------|
| =memory list [--symbol SYM] [--source user\vert{}reflector] [--all]= | 列出语义规则。=--source= 按来源筛选；=--all= 包含已禁用的规则。 |
| =memory add --rule TEXT [--symbol SYM] [--confidence 0.8]= | 添加语义规则。=--symbol= 为空表示全局规则。 |
| =memory edit --id N [--rule TEXT] [--confidence F]= | 编辑已有规则的文本或置信度。 |
| =memory delete --id N= | 删除规则。 |
| =memory toggle --id N= | 切换规则启用/禁用状态。 |
| =memory episodic [--symbol SYM] [--limit N] [--json]= | 查看 LLM 生成的交易反思记录，包含方向、入场/出场价、盈亏百分比和反思文本。 |

*** mcp — MCP 服务

通过 Model Context Protocol 将 brale-core 分析能力暴露给 AI Agent。

**** mcp serve — 启动 MCP 服务器

#+begin_src bash
# stdio 模式（用于 Claude Code / Claude Desktop）
bralectl mcp serve --mode stdio --system PATH --index PATH

# HTTP 模式（用于远程或容器化访问）
bralectl mcp serve --mode http --addr 0.0.0.0:8765 --system PATH --index PATH
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--mode= | =stdio= | 传输模式：=stdio= 或 =http= |
| =--addr= | =127.0.0.1:8765= | HTTP 模式监听地址 |
| =--system= | =configs/system.toml= | system.toml 路径 |
| =--index= | =configs/symbols-index.toml= | symbols-index.toml 路径 |
| =--audit-log= | 自动生成 | MCP 审计日志文件路径 |

**** mcp install — 自动写入 MCP 客户端配置

#+begin_src bash
bralectl mcp install --target claude-code
bralectl mcp install --target claude-desktop
bralectl mcp install --target opencode
bralectl mcp install --target codex
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--target= | =claude-code= | 安装目标：=claude-code=、=claude-desktop=、=opencode=、=codex= 或 =custom= |
| =--mode= | =http= | 安装模式：默认写入 Streamable HTTP 配置；=codex= 默认也使用 HTTP 容器端点 |
| =--config= | （自动检测） | 显式指定 MCP 配置文件路径 |
| =--command= | （自动检测） | bralectl 可执行文件路径 |
| =--name= | =brale-core= | MCP server 名称 |
| =--system= | =configs/system.toml= | system.toml 路径 |
| =--index= | =configs/symbols-index.toml= | symbols-index.toml 路径 |
| =--audit-log= | （空） | MCP 审计日志文件路径 |

*** test — E2E 测试

在隔离的 Docker 栈中执行 E2E 回归测试。

**** test list — 列出可用 suite

#+begin_src bash
bralectl test list
#+end_src

**** test run — 执行测试

#+begin_src bash
bralectl test run --suites ctl,decision --symbol SOLUSDT --timeout 15m
BRALE_E2E_TELEGRAM_TOKEN=123456:ABC... BRALE_E2E_TELEGRAM_CHAT_ID=-1001234567890 \
  bralectl test run --suites ctl,decision,reconcile
#+end_src

| 参数 | 默认值 | 说明 |
|------+---------+------|
| =--profile= | =quick-lifecycle= | E2E 配置 profile |
| =--suites= | =ctl= | 要执行的 suite，逗号分隔 |
| =--symbol= | =SOLUSDT= | 测试交易对 |
| =--ft-endpoint= | =http://127.0.0.1:18080= | Freqtrade API 地址 |
| =--pg-port= | =15432= | PostgreSQL 端口 |
| =--timeout= | =15m= | 全局超时 |
| =--force-close-after= | =5m= | 强制平仓等待时间 |
| =--report-dir= | =_output/e2e= | 报告输出目录 |
| =--bootstrap= | =auto= | 引导模式 (auto/strict) |
| =--telegram-token= | 环境变量 | Telegram Bot token；默认读 =BRALE_E2E_TELEGRAM_TOKEN= / =NOTIFICATION_TELEGRAM_TOKEN= |
| =--telegram-chat-id= | 环境变量 | Telegram chat_id；默认读 =BRALE_E2E_TELEGRAM_CHAT_ID= / =NOTIFICATION_TELEGRAM_CHAT_ID= |

** 使用示例

*** 日常运维

#+begin_src bash
bralectl schedule status                    # 查看调度状态
bralectl position list --json               # 当前持仓（JSON）
bralectl decision latest --symbol BTCUSDT   # 最新决策
bralectl config show                        # 运行配置摘要
#+end_src

*** 回测与验证

#+begin_src bash
# Gate 规则回放——用当前规则重放历史决策
bralectl backtest rules --symbol BTCUSDT \
  --from 2025-01-01 --to 2025-06-01 --format html --output report.html

# 指标引擎交叉校验
bralectl indicator diff --symbol BTCUSDT

# 离线配置验证
bralectl config validate
#+end_src

*** 记忆系统

#+begin_src bash
bralectl memory add --symbol BTCUSDT --rule "RSI > 80 时不做多" --confidence 0.9
bralectl memory list --source reflector      # 查看自动生成的规则
bralectl memory toggle --id 3                # 禁用某条规则
bralectl memory episodic --symbol BTCUSDT    # 查看交易反思
#+end_src

*** MCP 集成

#+begin_src bash
bralectl setup --install-mcp --mcp-target opencode  # 初始化并安装到 OpenCode
bralectl mcp install --target opencode       # 一键安装到 OpenCode
bralectl mcp serve --mode http --addr 0.0.0.0:8765  # 远程 HTTP 模式
#+end_src

*** LLM 诊断与币种管理

#+begin_src bash
bralectl llm probe                           # 探测所有 stage
bralectl llm probe --stage indicator         # 只探测 indicator stage
bralectl add-symbol ETHUSDT --dry-run        # 预览新币种配置
bralectl add-symbol SOLUSDT --template-symbol BTCUSDT  # 以 BTC 为模板
#+end_src

*** E2E 测试

#+begin_src bash
bralectl test list                           # 列出所有可用 suite
bralectl test run --suites ctl               # 快速回归
bralectl test run --suites ctl,decision,reconcile  # 多 suite 回归
BRALE_E2E_TELEGRAM_TOKEN=123456:ABC... BRALE_E2E_TELEGRAM_CHAT_ID=-1001234567890 \
  bralectl test run --suites ctl,decision,reconcile  # 测试完成后推送 Telegram 摘要
bralectl test run --suites lifecycle-force-close \
  --timeout 30m --ft-endpoint http://127.0.0.1:18080  # 完整生命周期
#+end_src

** Overview

=bralectl= is the command-line management tool for the brale-core quantitative trading
engine. It connects to a running brale-core instance via the Runtime API (default
=http://127.0.0.1:9991=) for operational tasks, and supports offline operations —
backtesting, indicator comparison, MCP serving — that require only config files and a
local database.

** Global Flags

| Flag | Default | Description |
|------+---------+-------------|
| =--endpoint= | =http://127.0.0.1:9991= | Runtime API address |
| =--json= | =false= | Emit machine-readable JSON output |

** Commands

*** llm — LLM Diagnostics

Probe configured LLM endpoints for structured output support.

#+begin_src bash
bralectl llm probe [--repo .] [--stage indicator|structure|mechanics]
#+end_src

| Flag | Default | Description |
|------+---------+-------------|
| =--repo= | =.= | Project root for config resolution |
| =--stage= | (empty — probe all) | Probe a single stage: =indicator=, =structure=, =mechanics= |

Example output:

#+begin_src text
indicator   OK    model=gpt-4o   endpoint=https://api.openai.com/v1
structure   OK    model=gpt-4o   endpoint=https://api.openai.com/v1
mechanics   FAIL  model=gpt-4o   endpoint=https://api.openai.com/v1  err=timeout
#+end_src

Evaluate recent LLM round metadata quality:

#+begin_src bash
bralectl llm eval [--symbol BTCUSDT] [--limit 20] [--db postgres://...]
#+end_src

| Flag | Default | Description |
|------+---------+-------------|
| =--symbol= | (empty) | Filter rounds by trading pair |
| =--limit= | =20= | Maximum number of rounds to evaluate |
| =--db= | (empty) | PostgreSQL DSN |

Inspect the prompt registry:

#+begin_src bash
bralectl llm prompts list [--role decision] [--active-only] [--db postgres://...]
#+end_src

| Flag | Default | Description |
|------+---------+-------------|
| =--role= | (empty) | Filter prompt entries by role |
| =--active-only= | =false= | Show only active prompt entries |
| =--db= | (empty) | PostgreSQL DSN |

*** indicator — Indicator Engine Tools

Compare indicator outputs between different engines. Available: =ta= (default, pure Go), =talib= (CGO), =reference= (legacy pure Go).

#+begin_src bash
bralectl indicator diff --symbol BTCUSDT \
  [--index PATH] [--baseline talib] [--candidate ta] [--format text|json]
#+end_src

| Flag | Default | Description |
|------+---------+-------------|
| =--symbol= | (required) | Trading pair |
| =--index= | =configs/symbols-index.toml= | Symbol index path |

| =--candidate= | =ta= | Comparison engine: =ta=, =talib= or =reference= |
| =--format= | =text= | Output format: =text= or =json= |

*** backtest — Gate Rule Replay

Replay historical decision rounds through the current Gate/Sieve rule engine without
invoking any LLM calls.

#+begin_src bash
bralectl backtest rules --symbol BTCUSDT \
  --from 2025-01-01 --to 2025-06-01 \
  [--format text|json|html] [--output report.html] \
  [--system PATH] [--index PATH] [--db PATH]
#+end_src

| Flag | Default | Description |
|------+---------+-------------|
| =--symbol= | (required) | Trading pair |
| =--from= / =--to= | (required) | Date range — =YYYY-MM-DD= or RFC 3339 |
| =--system= | =configs/system.toml= | System config path |
| =--index= | =configs/symbols-index.toml= | Symbol index path |
| =--db= | from system config | PostgreSQL DSN |

*** memory — Memory System Management

Manage the semantic rule store and view episodic trade reflections.
All memory subcommands accept =--db= to specify the PostgreSQL DSN.
There is currently no =--system= flag here; when omitted, the command falls back to a local default DSN.

| Subcommand | Description |
|------------+-------------|
| =memory list [--symbol SYM] [--source user\vert{}reflector] [--all]= | List semantic rules. =--source= filters by origin; =--all= includes disabled rules. |
| =memory add --rule TEXT [--symbol SYM] [--confidence 0.8]= | Create a semantic rule. Omit =--symbol= for a global rule. |
| =memory edit --id N [--rule TEXT] [--confidence F]= | Update an existing rule's text or confidence. |
| =memory delete --id N= | Remove a rule. |
| =memory toggle --id N= | Toggle a rule's enabled/disabled state. |
| =memory episodic [--symbol SYM] [--limit N] [--json]= | View LLM-generated trade reflections with direction, prices, PnL %, and reflection text. |
