*** Dashboard

| 路径 | 说明 |

| =/api/runtime/dashboard/kline= | K 线与关键点位 |
| =/api/runtime/dashboard/decision_flow= | 决策链路图 |
| =/api/runtime/dashboard/decision_history= | 历史决策列表与详情 |
| =/api/runtime/dashboard/account_summary= | 账户概要（余额、持仓、盈亏汇总） |

**** =/api/runtime/dashboard/decision_history=

- 必填查询参数：=symbol=
- 可选查询参数：
  - =limit= — 每页条数，默认 20
  - =cursor= — 来自上一页 =next_cursor= 的 keyset 游标
  - =snapshot_id= — 指定要查看详情的快照
- 返回字段：
  - =items= — 当前页历史决策
  - =next_cursor= — 下一页游标；为空表示没有更多数据
  - =detail= — 当传入 =snapshot_id= 时返回该快照详情

分页语义：

1. 历史列表基于 =created_at DESC, id DESC= 做稳定 keyset 排序
2. 客户端应将 =next_cursor= 原样透传到下一次请求
3. Dashboard 前端使用“加载更多”增量请求，而不是单次放大 =limit=

** 通知

支持以下出站通知渠道：

- Telegram Bot — 支持决策推送、命令交互（如 /observe）
- 飞书 Bot — 支持决策推送
- Email Sender — 通过 SMTP 发送 HTML / 文本告警（需手工配置 =[notification.email]=）

系统启动后自动拉起已配置的 Bot。启动通知由 =startup_notify_enabled= 控制。

通知类型：
- 决策报告（图片卡片）— 每轮决策后推送
- 开仓通知（文字消息）— 开仓时推送，包含入场价、止损、止盈
- 平仓通知（文字消息）— 平仓时推送，重复通知自动合并
- 部分平仓通知（文字消息）— 减仓时推送
- 风控更新通知（文字消息）— 止损/止盈调整时推送
- 启动通知（文字消息）— 系统就绪后推送，含币种、周期、余额
- 停止通知（文字消息 / HTML）— 系统关闭时推送，含运行时长
- 错误通知（文字消息 / HTML）— 严重错误时推送至配置的通知渠道

** Runtime API

Routes are defined in =internal/transport/runtimeapi/server.go=.

*** Dashboard

| Path | Description |
|------+-------------|
| =/api/runtime/dashboard/overview= | Dashboard summary cards |
| =/api/runtime/dashboard/kline= | Candles and key price markers |
| =/api/runtime/dashboard/decision_flow= | Decision flow graph |
| =/api/runtime/dashboard/decision_history= | Decision history list and detail |
| =/api/runtime/dashboard/account_summary= | Account summary (balance, positions, P&L) |

**** =/api/runtime/dashboard/decision_history=

- Required query parameter: =symbol=
- Optional query parameters:
  - =limit= — page size, defaults to 20
  - =cursor= — keyset cursor returned as =next_cursor=
  - =snapshot_id= — snapshot whose detail should be loaded
- Response fields:
  - =items= — current page of decision history
  - =next_cursor= — cursor for the next page; empty when exhausted
  - =detail= — snapshot detail when =snapshot_id= is provided

Pagination semantics:

1. The list uses stable keyset ordering on =created_at DESC, id DESC=
2. Clients should pass =next_cursor= through unchanged on the next request
3. The dashboard UI appends pages with “load more” instead of inflating =limit=

** Notifications

The following outbound notification channels are supported:

- Telegram Bot — decision push and command interaction (e.g., /observe)
- Feishu Bot — decision push
- Email Sender — SMTP-based HTML / text alerts (requires manual =[notification.email]= config)

Configured bots start automatically on launch. Startup notifications are controlled by =startup_notify_enabled=.

Notification types:
- Decision report (image card) — sent after each decision round
- Position open (text message) — sent on entry, showing entry price, stop loss, take profit
- Position close (text message) — sent on full close, duplicate notifications merged
- Partial close (text message) — sent on partial exit
- Risk update (text message) — sent on stop loss / take profit adjustment
- Startup (text message) — sent after system initialization, includes symbols, intervals, balance
- Shutdown (text message / HTML) — sent on graceful stop with uptime
- Error (text message / HTML) — critical errors pushed to configured channels
