** Brale 是什么

Brale 是一个 AI 驱动的量化交易引擎。它把一轮交易决策拆成 Observe → Decide → Plan → Execute → Monitor → Reconcile 六个阶段，由多个 LLM Agent 分别负责市场分析，Go 代码负责风控约束和执行调度，Freqtrade 负责最终下单。

| 阶段 | 做什么 |
|------+--------|
| Observe | 采集 K 线、技术指标、市场结构、衍生品与情绪数据 |
| Decide | 多路 Agent 独立分析，规则引擎做共识判定 |
| Plan | 生成风险计划：入场价、止损、止盈、仓位 |
| Execute | Freqtrade 执行下单 |
| Monitor | 实时监控仓位风险 |
| Reconcile | webhook / 轮询做状态对账 |

** 核心特点

- *LLM 出主意，代码做裁判* — 模型给出止盈止损方案后，Go 代码会强制校验价格序列、杠杆上限、清算安全距离。模型可以犯错，代码不让它真的亏钱。
- *三路 Agent 独立决策* — 指标分析、市场结构、衍生品力学各自独立运行，通过共识阈值避免单一模型误判。
- *全自动调度* — 定时决策、价格监控、状态对账都由内置调度器完成，不需要外部 cron。
- *执行层可替换* — 当前对接 Freqtrade，接口解耦，可以换成其他交易后端。
- *通知开箱即用* — 内置 Telegram 和飞书 Bot。

** 快速启动

*前置条件：* Docker、Docker Compose。本地开发额外需要 Go 1.25+。

#+begin_src bash

#+begin_src bash

cp .env.example .env

不需要引导页的话，直接 =make start= 启动 Freqtrade 和 Brale。

* English Version

** What Is Brale

Brale is an AI-driven quantitative trading engine. It breaks a trading decision into six stages — Observe → Decide → Plan → Execute → Monitor → Reconcile — where multiple LLM Agents handle market analysis, Go code enforces risk constraints and scheduling, and Freqtrade places the actual orders.

| Stage | What It Does |
|-------+--------------|
| Observe | Collect candlesticks, indicators, market structure, derivatives and sentiment data |
| Decide | Independent Agents analyze in parallel; rule engine reaches consensus |
| Plan | Generate a risk plan: entry, stop-loss, take-profit, position size |
| Execute | Freqtrade places the order |
| Monitor | Real-time position risk monitoring |
| Reconcile | State sync via webhooks and polling |

** Core Features

- *LLM proposes, code disposes* — after the model suggests TP/SL levels, Go code enforces price sequence validity, leverage caps, and liquidation safety. The model can be wrong; the code won't let it lose real money.
- *Three independent Agent lanes* — indicator analysis, market structure, and derivatives mechanics each run separately. Consensus thresholds prevent single-model misjudgment.
- *Fully automated scheduling* — decision cycles, price monitoring, and state reconciliation are all handled by a built-in scheduler. No external cron needed.
- *Swappable execution layer* — currently wired to Freqtrade, but the interface is decoupled and can be replaced with another trading backend.
- *Notifications out of the box* — built-in Telegram and Feishu bots.
