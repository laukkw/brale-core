** 调整优势

1. *LLM 韧性大幅提升* — 熔断器按模型隔离故障半径；令牌预算防止单轮失控；递归 Schema 保护杜绝无限展开；重试日志从 =nil= 升级为结构化 Logger，故障排查有据可查。

2. *CLI-first 工作流* — 废弃浏览器依赖的 Web Onboarding，所有初始化、配置生成和 MCP 安装均通过 =bralectl= 命令完成。服务器环境无需桌面即可全流程部署。

3. *MCP Streamable HTTP* — 替代 SSE 单向流，支持双向通信；Docker 探针保障容器间连通性；=bralectl mcp install= 自动配置客户端。

4. *通知系统精简* — Notice 类型体系统一；术语翻译全部在 Go Payload Builder 层完成，OG 卡片渲染层仅负责排版；新增关机 / 错误 / Gate Plan 三类卡片，覆盖运维场景。

5. *Tighten 风控增强* — 新增 =action= (adjust/hold) 与 =reason= 字段，LLM 可选择"维持现状"而非强制调整；Dashboard 合约同步更新，前端可展示拒绝原因。

6. *Docker 栈可隔离* — Compose 参数化后，主栈与 E2E 栈可在同一宿主机共存，端口和容器名互不冲突。PostgreSQL 数据独立挂载，备份更清晰。

7. *文档体系补全* — 同步子系统指南、字段翻译参考、部署与排障指南；本地 .docs 工作区作为开发辅助材料，不再写成已发布产物。

8. *测试覆盖扩展* — TA 边界用例、OG 卡片渲染、递归 Schema、Prompt 容错等新增测试，加固回归安全网。
